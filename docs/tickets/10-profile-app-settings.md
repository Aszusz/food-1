# 10 - Profile and App Settings

**Type:** AFK
**Blocked by:** [02 - Household Workspace Navigation](02-workspace-navigation.md)

## What To Build

Complete the non-household Settings surfaces from the information architecture. Members can open Profile, update their display name, and log out. App Settings is reachable and clearly states that there are no app preferences yet; do not invent preference features. Logging out ends the session and prevents protected workspace access until the user logs in again.

## Acceptance Criteria

- [ ] Settings links to Household, Profile, and App Settings.
- [ ] A member can update their display name and sees the saved name after refresh.
- [ ] Log Out ends the session and returns the user to Log In.
- [ ] A logged-out user cannot reopen a protected workspace URL.
- [ ] App Settings is reachable and displays its empty state.
